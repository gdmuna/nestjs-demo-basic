# Harness Engineering 文档规范

本文档继承 [docs/STANDARD.md](../STANDARD.md)，定义 `docs/02-harness/` 目录的专项约束。

`docs/02-harness/` 目录存放**Harness Engineering**理念相关的文档，回答"Harness Engineering 是什么、为什么、怎么做"。

---

## 1. 文档定位约束

Harness Engineering 文档具有以下特征：

- **理念导向**：解释驾驭层的概念与价值，不写具体功能操作步骤（操作步骤放 `01-guides/`）
- **面向读者**：目标读者是想理解"为什么这个项目要这样设计 AI 协作机制"的开发者
- **与实现绑定**：每个驾驭层机制必须在本项目中有对应的真实实现（不写空洞的理论）
