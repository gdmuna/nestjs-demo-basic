# Docusaurus 文档站快速上手

本指南指导你在本项目中接入 **Docusaurus v3**，将现有 `docs/` 目录渲染为一个本地可预览、可构建部署的文档站。

> 目标：最小可用（MVP）配置。不包含博客、国际化、版本控制等高级特性。

---

## 快速配置清单

完成以下所有步骤后，文档站即达到最小可用状态：

- [ ] Node.js ≥ 20（推荐 22）
- [ ] 在项目根目录执行脚手架命令，生成 `website/` 子目录
- [ ] 修改 `website/docusaurus.config.ts`：指向 `../docs`、关闭 blog、启用 docs-only 模式
- [ ] 修改 `website/sidebars.ts`：使用 autogenerate 自动生成侧边栏
- [ ] 排除 `docs/` 中不需要渲染的元文件（AGENTS.md、STANDARD.md、.obsidian/）
- [ ] 确认 `docs/README.md` 添加 `slug: /` 成为首页
- [ ] 在根目录 `package.json` 添加 `docs:dev` / `docs:build` 快捷脚本
- [ ] 运行 `pnpm docs:dev` 验证本地预览正常

---

## 分步操作

### 第一步：脚手架生成 website/ 子项目

在项目**根目录**执行：

```bash
pnpm dlx create-docusaurus@latest website classic --typescript --package-manager pnpm
```

> 说明：`website/` 是文档站的独立子目录，与 NestJS 应用代码解耦。`classic` 模板内置暗黑模式和搜索支持。

执行后目录结构：

```
nestjs-demo-basic/
├── docs/          ← 现有文档源文件（不动）
├── website/       ← 新生成的 Docusaurus 子项目
│   ├── blog/      ← 可以直接删除
│   ├── docs/      ← 可以直接删除（用 ../docs 替代）
│   ├── src/
│   ├── static/
│   ├── docusaurus.config.ts
│   ├── sidebars.ts
│   └── package.json
└── package.json
```

**删除不需要的目录：**

```bash
Remove-Item -Recurse -Force website/blog
Remove-Item -Recurse -Force website/docs
```

---

### 第二步：配置 docusaurus.config.ts

将 `website/docusaurus.config.ts` 替换为以下内容：

```ts
import type { Config } from '@docusaurus/types';
import type * as Preset from '@docusaurus/preset-classic';

const config: Config = {
  title: 'NestJS Demo Basic',
  tagline: 'NestJS 后端开发基线模板',
  favicon: 'img/favicon.ico',

  // 部署时替换为实际域名
  url: 'https://your-domain.com',
  baseUrl: '/',

  onBrokenLinks: 'warn',
  onBrokenMarkdownLinks: 'warn',

  i18n: {
    defaultLocale: 'zh-Hans',
    locales: ['zh-Hans'],
  },

  presets: [
    [
      'classic',
      {
        docs: {
          // 指向项目根目录的 docs/ 文件夹
          path: '../docs',
          // docs-only 模式：文档直接挂载到根路径 /
          routeBasePath: '/',
          sidebarPath: './sidebars.ts',
          // 排除元文件，不将其渲染为文档页面
          exclude: [
            '**/.obsidian/**',
            '**/AGENTS.md',
            '**/STANDARD.md',
          ],
        },
        // 关闭博客功能
        blog: false,
        theme: {
          customCss: './src/css/custom.css',
        },
      } satisfies Preset.Options,
    ],
  ],

  themeConfig: {
    navbar: {
      title: 'NestJS Demo Basic',
      logo: {
        alt: 'Logo',
        src: 'img/logo.svg',
      },
      items: [
        {
          type: 'docSidebar',
          sidebarId: 'mainSidebar',
          position: 'left',
          label: '文档',
        },
        {
          href: 'https://github.com/your-org/nestjs-demo-basic',
          label: 'GitHub',
          position: 'right',
        },
      ],
    },
    footer: {
      style: 'dark',
      links: [],
      copyright: `Copyright © ${new Date().getFullYear()} NestJS Demo Basic. Built with Docusaurus.`,
    },
    prism: {
      additionalLanguages: ['bash', 'json', 'typescript', 'docker'],
    },
  } satisfies Preset.ThemeConfig,
};

export default config;
```

---

### 第三步：配置 sidebars.ts

将 `website/sidebars.ts` 替换为以下内容，使用自动生成侧边栏：

```ts
import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  mainSidebar: [
    {
      type: 'autogenerated',
      // 从 ../docs（即 path 指向的目录）自动生成
      dirName: '.',
    },
  ],
};

export default sidebars;
```

> autogenerate 会根据 `docs/` 的目录结构自动生成分类和文档链接，无需手动维护。

---

### 第四步：设置文档首页

docs-only 模式下，根路径 `/` 需要一个对应的文档页面。编辑 `docs/README.md`，在 frontmatter 中添加 `slug: /`：

```markdown
---
title: 文档索引
slug: /          # ← 添加此行，使其成为首页
status: active
...
---
```

---

### 第五步：在根目录添加快捷脚本

编辑根目录 `package.json`，在 `scripts` 中添加：

```json
{
  "scripts": {
    "docs:dev":   "cd website && pnpm start",
    "docs:build": "cd website && pnpm build",
    "docs:serve": "cd website && pnpm serve"
  }
}
```

---

### 第六步：安装依赖并启动

```bash
cd website
pnpm install
cd ..

# 启动预览（默认 http://localhost:3000）
pnpm docs:dev
```

## 常见问题

**Q：某些文档页面提示 broken link？**
A：`AGENTS.md` / `STANDARD.md` 中的相对链接在 Docusaurus 环境下会失效，这是预期行为（`onBrokenLinks: 'warn'` 设为警告而非报错）。

**Q：frontmatter 字段（如 `status`、`category`）在页面上显示了？**
A：Docusaurus 只识别它知道的 frontmatter 字段，未知字段默认被忽略，不会显示。如果确实渲染出来，检查 Markdown 格式是否规范。

**Q：侧边栏排序不符合预期？**
A：在各目录下添加 `_category_.json` 控制排序和标签，例如：

```json
{
  "label": "架构设计",
  "position": 2,
  "collapsed": false
}
```

**Q：想添加全文搜索？**
A：最简单的方案是安装本地搜索插件（无需 Algolia 账号）：

```bash
cd website
pnpm add @easyops-cn/docusaurus-search-local
```

然后在 `docusaurus.config.ts` 的 `themes` 中添加：

```ts
themes: [
  [
    require.resolve('@easyops-cn/docusaurus-search-local'),
    { hashed: true, language: ['zh', 'en'] },
  ],
],
```
