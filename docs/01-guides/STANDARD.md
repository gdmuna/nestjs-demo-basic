# 指南写作规范

本文档继承 [docs/STANDARD.md](../STANDARD.md)，定义 `docs/01-guides/` 目录的专项约束。

---

## 1. 指南文档要求

指南面向**人类开发者**，回答"怎么做？"，具有以下特征：

- **面向任务**：每篇指南围绕一个具体操作目标（如"配置开发环境"、"添加新模块"）
- **步骤化**：使用有序列表呈现操作步骤，每步一个明确动作
- **可验证**：每个关键步骤后提供验证方法（如"运行 `xxx` 命令，预期输出为 `yyy`"）
- **前置条件**：在文档开头声明读者需要的环境、权限或前置知识

---

## 2. 结构模板

```markdown
# 指南标题

> 前置条件：……

## 步骤

1. 第一步
2. 第二步
   - 验证：`命令` → 预期输出

## 常见问题
```
